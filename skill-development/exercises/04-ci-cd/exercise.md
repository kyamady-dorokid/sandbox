# CI/CD & 自動テスト 演習（MVP）

## 概要
マスタープラン「生成AI・CI/CDパイプラインを用いた自動テスト基礎」に従った実装演習。
GitHub Actions を用いて自動テストパイプラインを構築し、リポジトリへの push 時にテストが自動実行される環境を整備する。

## 学習ゴール
- ユニットテスト（JUnit または pytest）を書き、ローカルで実行できる
- GitHub Actions ワークフロー（YAML）を理解し、カスタマイズできる
- テスト実行結果をレポートに出力できる
- カバレッジ測定の基本を理解する

## 期待する成果物
- テストコード（対象言語別、最低 3 ケース以上）
- `.github/workflows/` 内の CI パイプライン定義（YAML）
- テスト実行結果のレポート（例: HTML カバレッジレポート）

## ワークフロー設計（最小例）
```
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-<language>@v3
      - run: # 依存関係をインストール
      - run: # テストを実行
      - run: # カバレッジレポート（任意）
```

## 前提環境
- リポジトリに対象言語（Java/Python 等）のテストコードが存在すること

## セットアップ手順（最小例）
1. リポジトリのルートに `.github/workflows/ci.yml` を作成
2. ワークフロー YAML にテスト実行コマンドを記載
3. git push でワークフローが自動実行される
4. GitHub の Actions タブで実行結果を確認

## テスト要件
- 対象プロジェクトの既存コードに対して最低 3 つ以上のテストケースを追加
- テストが CI 上で自動実行されることを確認
- カバレッジ測定（任意だが推奨）

## 完了条件
- [ ] ローカルでテストが成功する
- [ ] GitHub Actions ワークフローが定義されている
- [ ] push 時にワークフローが自動実行される
- [ ] テスト実行結果が Actions タブに表示される

## 推定所要時間
- 仕様作成: 3〜6 時間（既に完了）
- 実装: 4〜8 時間

## 備考
