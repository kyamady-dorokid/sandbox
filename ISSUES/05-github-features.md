## 概要
このIssueは「GitHub機能の一覧リスティングと概要把握」項目の演習仕様を作成するタスクです。チーム開発で有用な GitHub の主要機能をハンズオン形式で学べるように設計します。

## 学習ゴール
- Git とブランチ戦略の標準的なワークフローを説明・実践できる
- Pull Request と Code Review の流れを理解し運用できる
- GitHub Actions / Issues / Projects / Security 機能の使いどころを判断できる

## 前提条件
- Git の基本操作ができること（clone, commit, push, branch）

## 成果物（Acceptance Criteria）
- `exercise.md` を作成し、このIssue に添付/リンクすること
- 複数の GitHub 機能を組み合わせた実践ワークフローの設計（例: PR -> CI -> Review -> Merge）を提示すること

## スコープ（MVP）
- 基本的な PR ワークフローと GitHub Actions を交えた CI の流れを理解・実践できること

## 作業タスク
1. `exercise.md` をテンプレートから作成
2. ワークフロー図または手順を記載（PR フロー、ブランチ命名規則、Review 基準等）
3. 実践演習（Issue を作る、PR を作る、Action を触る）のチェックリストを作成

## 推定作業時間
- 仕様作成: 2〜4 時間

## チェックリスト
- [ ] `exercise.md` を作成して添付/リンクする
- [ ] ワークフロー手順を記載する
