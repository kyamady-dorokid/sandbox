## 概要
このIssueは「生成AI・CI/CDパイプラインを用いた自動テスト基礎」項目の演習仕様を設計するタスクです。GitHub Actions を用いた自動テストパイプラインの最低構成を定義します。

## 学習ゴール
- ユニットテストと統合テストの違いを理解し、テストを書く習慣を身につける
- GitHub Actions を用いた基本的な CI パイプライン（テストの実行、カバレッジ計測）を構築できる
- 生成AI を活用したテストケース生成の導入方法を理解する

## 前提条件
- 対象リポジトリにテスト対象のサンプルコード（Java/Python 等）が存在すること（またはスターターコードを用意すること）

## 成果物（Acceptance Criteria）
- `exercise.md` を作成し、このIssue に添付/リンクすること
- GitHub Actions ワークフローのサンプルを示し、ローカル or CI 上でテストが回ることを確認する手順を記載すること

## スコープ（MVP）
- 最低限: 単体テストが CI 上で実行され、結果が表示されること
- 追加: カバレッジレポートの生成、簡単な静的解析（例: flake8, spotbugs）はオプション

## 作業タスク
1. `exercise.md` をテンプレートから作成
2. 対象言語/フレームワーク別のワークフロー例（Java: Maven, Python: pytest）を用意
3. カバレッジ測定とレポート出力の方針を記載
4. 生成AI を使ったテスト生成のサンプル手順（オプション）を記載

## 推定作業時間
- 仕様作成: 3〜6 時間

## チェックリスト

- [ ] GitHub Actions ワークフローのサンプルを記載する
- [ ] テスト実行手順を明記する
